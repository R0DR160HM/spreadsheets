# Spreadsheet

An offline-first PWA spreadsheet built with Angular 21 (standalone components, signals, zoneless).

## Features

- **Rich text cells** — mixed formatting inside a single cell (bold, italic, underline, strikethrough, color, font size), plus cell background and alignment. Edit with `Enter`/`F2`/double-click; `Alt+Enter` inserts a line break inside a cell.
- **Formulas** — start a cell with `=` for Excel-style math: `+ - * / ^`, parentheses, cell references (`=A1*B2+1`, typed or inserted by clicking a cell mid-formula), ranges and functions (`SUM`, `AVERAGE`, `MIN`, `MAX`, `COUNT`, `ABS`, `ROUND`). Errors show as `#ERROR!`, `#CIRC!`, `#DIV/0!`, `#VALUE!`. Formulas stay live through XLSX; CSV and PDF export the computed values.
- **Multi-cell selection** — drag, `Shift`+click or `Shift`+arrows to select a range; formatting and `Delete` apply to the whole range.
- **Mobile-friendly** — responsive layout with touch-sized controls; tap to select, double-tap to edit.
- **Undo/redo** — the last 100 changes, via `Ctrl+Z` / `Ctrl+Y`.
- **Multiple sheets** — tabs at the bottom (double-click to rename).
- **Column resizing** — drag the handle at a column header's edge (or focus it and use arrow keys). Widths round-trip through XLSX.
- **File formats** — CSV (with a column-separator prompt) and XLSX import/export, PDF export, through the *File* menu.
- **Offline** — works fully offline via an Angular service worker; the open workbook autosaves to IndexedDB.
- **Internationalization** — Portuguese when the system language is `pt*`, English otherwise.

## Adding a file format

Formats are plugins registered in `src/app/core/formats/format-registry.service.ts`. Add a descriptor whose `load()` lazily imports a class implementing `SpreadsheetFormat` (`read` for import, `write` for export, either or both):

```ts
this.register({
  id: 'ods',
  label: 'ODS',
  extension: 'ods',
  mimeType: 'application/vnd.oasis.opendocument.spreadsheet',
  canImport: true,
  canExport: true,
  load: () => import('./ods-format').then((m) => new m.OdsFormat()),
});
```

The File menu, the open dialog's file filter, and the format's lazy chunk all follow automatically. A descriptor may also declare `options` (e.g. CSV's separator) — the app prompts the user before reading/writing.

## Development

```bash
npm start          # dev server (NO service worker — not installable/offline)
npm test           # unit tests (vitest)
npm run build      # production build (dist/spreadsheet/browser)
npm run preview    # build + serve the production build at http://localhost:4200
npm run serve:dist # serve an existing production build
```

## Installing the PWA / offline use

Offline support comes from the Angular service worker, which **only exists in
production builds** — `ng serve` never registers it, so a PWA installed from
the dev server cannot work offline.

1. `npm run preview`
2. Open http://localhost:4200 once and install the app from there.
3. That's it — after that first load the service worker has prefetched every
   asset (including the lazy CSV/XLSX/PDF chunks), and the app works with the
   server stopped. Your workbook autosaves to IndexedDB, so content survives
   restarts offline too.
