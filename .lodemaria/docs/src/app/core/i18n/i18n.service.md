# i18n-service

## Purpose
The `I18nService` is a service that handles the translation of user interface messages. It detects the system's locale and uses it to determine which translation to use from an external translations object.

## Public/Exported Class, Function, Constant, Entry Point
- **`locale`:** A signal that tracks the current locale.
- **`t(key: TranslationKey, params?: Record<string, string | number>): string`:** Translates a key with optional parameters using the detected locale.
  - Parameters are replaced according to `params`.
  - **Error Handling:** Throws an error if the translation cannot be found in the provided translations object.

## Notable Internal Logic
1. **Language Detection:** The function detects the system's locale from `navigator.language` and converts it to lowercase using `startsWith('pt')`. This ensures that all keys are in Portuguese.
2. **Translation Handling:** It uses the detected locale to retrieve the appropriate translation from the `TRANSLATIONS` object. If the key is not found, an error is thrown.

## When Several Companion Files are Given
Each companion file is included as a separate unit within the same project or as part of a larger package with their own source code. This setup allows for easier management and maintenance of different parts of the application.
